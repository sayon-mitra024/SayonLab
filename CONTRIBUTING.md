# Contributing to SayonLab

Thank you for your interest in contributing to SayonLab.

SayonLab is currently in its early development stage (v0.1.x). During this phase, the project's architecture and public APIs are still evolving. As a result, we are not yet accepting large external code contributions.

## How You Can Help

Your feedback is highly appreciated. You can contribute by:

- Reporting bugs
- Suggesting new features
- Improving documentation
- Sharing ideas for scientific workflows
- Opening discussions about research-oriented functionality

Please use GitHub Issues for bug reports, feature requests, or general discussions.

## Code Contributions

At this stage, pull requests introducing major new features may not be accepted because the project structure is still being stabilized.

Once SayonLab reaches a more mature release, a detailed contribution guide, development workflow, coding standards, and pull request process will be added.

## Reporting Bugs

When reporting a bug, please include:

- Operating System
- Python version
- SayonLab version
- Steps to reproduce
- Expected behavior
- Actual behavior
- Error messages (if any)

## Feature Requests

Feature requests are welcome, especially those that improve scientific research productivity, reproducibility, publication-quality visualization, or researcher workflows.

Thank you for helping improve SayonLab.
